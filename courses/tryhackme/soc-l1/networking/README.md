# Networking

Sekcja zawiera notatki oraz ćwiczenia związane z podstawami sieci komputerowych, analizą ruchu i monitorowaniem bezpieczeństwa sieciowego z perspektywy analityka SOC.

## Kategorie

| Sekcja                                                                                                   | Zakres materiału                                                                                           |
| -------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| [Networking Basics](courses/tryhackme/soc-l1/networking/basics/README)                                   | Podstawy komunikacji sieciowej, adresacja IP, porty, protokoły oraz najważniejsze pojęcia sieciowe.        |
| [Network Security Monitoring](courses/tryhackme/soc-l1/networking/network-security-monitoring/README.md) | Monitorowanie bezpieczeństwa sieci, systemy IDS, Snort, wykrywanie ataków i analiza incydentów sieciowych. |
| [Network Traffic Analysis](courses/tryhackme/soc-l1/networking/network-traffic-analysis/README)          | Analiza pakietów i przepływów sieciowych przy użyciu narzędzi takich jak Wireshark i `tcpdump`.            |
| [Web Security Monitoring](courses/tryhackme/soc-l1/networking/web-security-monitoring/README)            | Monitorowanie ruchu HTTP/HTTPS, analiza logów serwerów WWW oraz wykrywanie ataków na aplikacje webowe.     |
## Główne zagadnienia

- modele OSI i TCP/IP,
- adresacja IPv4 i subnetting,
- protokoły i porty sieciowe,
- analiza ruchu sieciowego,
- filtrowanie pakietów,
- Network Security Monitoring,
- systemy IDS i reguły detekcji,
- analiza ruchu HTTP/HTTPS,
- wykrywanie anomalii i zagrożeń,
- analiza incydentów sieciowych.

## Cel sekcji

Celem materiałów jest rozwijanie umiejętności potrzebnych do:

1. rozumienia komunikacji pomiędzy hostami,
2. identyfikowania używanych protokołów i usług,
3. analizowania przechwyconego ruchu sieciowego,
4. wykrywania podejrzanych połączeń i anomalii,
5. interpretowania alertów IDS,
6. odtwarzania przebiegu incydentów na podstawie danych sieciowych.
